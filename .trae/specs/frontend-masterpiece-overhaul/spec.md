# Amphoreus 前端全面重构 - Product Requirement Document

## Overview
- **Summary**: 对 Amphoreus AI 故事引擎的前端进行彻底的视觉重构和体验升级。基于现有"The Manuscript Room"（手稿室）主题，打造一个具有沉浸感的、编辑级品质的创作工具界面，将当前基础的UI系统升级为具有独特美学个性、精致动效、层次丰富的高端应用体验。
- **Purpose**: 当前前端虽然已有基础的设计系统和组件，但在视觉深度、动效质感、交互体验、氛围营造上仍有巨大提升空间。需要将其打造为一个令人难忘的、具有"古籍手稿馆"氛围的创作空间，而不是普通的功能性SaaS仪表盘。
- **Target Users**: 小说作者、剧本作家、世界观构建者、故事创作者。

## Goals
- 打造一个极具辨识度、令人难忘的"古籍手稿馆"沉浸式视觉主题，超越普通工具类应用的审美
- 建立完整、精致、层次丰富的设计系统（Design System），包括更细腻的阴影、光效、纹理、边框等视觉语言
- 实现流畅、有意义的页面过渡和微交互动效，增强创作过程的仪式感
- 优化所有页面的布局、间距、排版，提升信息密度的同时保持优雅的呼吸感
- 强化侧栏导航的视觉层次和标识性，打造品牌感强的侧边栏体验
- 为核心创作页面（写作、角色、世界构建、情节、质量分析）注入场景化的视觉氛围
- 提升所有组件的视觉质感：按钮、输入框、卡片、模态框、进度条、通知等
- 优化响应式体验，确保在不同尺寸下都保持高品质视觉呈现

## Non-Goals (Out of Scope)
- 不改变后端API接口和业务逻辑
- 不修改核心功能流程和用户操作路径
- 不引入新的大型框架或重写整个应用架构
- 不改变路由结构和页面组织方式
- 不处理后端代码问题
- 不新增功能模块（除视觉效果增强相关外）

## Background & Context
- Amphoreus 是一个AI驱动的故事创作引擎，涵盖从世界构建、角色塑造、情节设计、场景生成到最终写作和质量审稿的全流程
- 现有主题为"The Manuscript Room"，使用暖墨色调（ink-bg/parchment/chop朱红印泥）和衬线字体，有很好的基础概念
- 当前使用 Vue 3 + TypeScript + Vite + Tailwind CSS v4 + ECharts + D3.js
- 已有基础组件类（btn, card, input, chip等），但视觉层次较平，质感不足
- 现有纸张纹理过于微弱，缺乏真实感和深度
- 动效较少且单一，缺乏叙事性的过渡体验
- 页面布局较为传统，缺少打破常规的视觉亮点

## Functional Requirements
- **FR-1**: 升级设计token系统，增加更丰富的颜色层次（高光、暗部、光晕、金属质感等）、阴影层级、动效曲线
- **FR-2**: 重构全局视觉氛围：增强纸张纹理、添加边缘光效、墨色渐变、微妙的图案装饰
- **FR-3**: 重写侧栏导航（AppLayout）：增强品牌标识区、优化导航项交互、添加精致的装饰元素、改进底部控制区
- **FR-4**: 升级所有基础组件的视觉样式：按钮（多种质感）、输入框（焦点状态增强）、卡片（悬停深度、边框光效）、标签/徽章、进度条、模态框
- **FR-5**: 优化Toast通知系统：更精致的入场出场动效、更优雅的图标和色彩
- **FR-6**: 增强页面过渡动效：页面切换时的叙事性转场、内容的阶梯式入场
- **FR-7**: 为各个页面添加场景化的装饰元素和视觉锚点
- **FR-8**: 优化ProjectsView首页：创建一个有仪式感的欢迎/项目选择界面
- **FR-9**: 升级WriterView写作界面：增强写作区域的沉浸感，优化排版和预览体验
- **FR-10**: 优化QualityView质量分析界面：让数据可视化更优雅，报告卡片更有设计感
- **FR-11**: 增强WorldBuilderView、CharacterView、PlotView、SceneView等工作台页面的视觉层次和交互质感
- **FR-12**: 优化SandboxView、InteractiveView、PipelineView的视觉体验
- **FR-13**: 完善浅色（Paper）主题，确保两个主题都达到同等品质
- **FR-14**: 优化ChatPanel对话面板，增强对话气泡的质感和加载动画

## Non-Functional Requirements
- **NFR-1**: 性能 - 所有动效和视觉增强不能显著影响帧率（保持60fps），不能增加超过50KB的CSS体积
- **NFR-2**: 可访问性 - 保持足够的颜色对比度，支持键盘导航，尊重reduced-motion设置
- **NFR-3**: 响应式 - 在1280px、1440px、1920px宽度下都有良好表现，平板尺寸下侧栏可折叠
- **NFR-4**: 主题一致性 - 所有页面必须遵循统一的设计语言，不能有视觉断层
- **NFR-5**: 浏览器兼容 - 支持现代浏览器（Chrome 100+, Firefox 100+, Safari 15+）
- **NFR-6**: 代码质量 - 保持TypeScript类型安全，CSS组织清晰可维护

## Constraints
- **Technical**: 必须使用Vue 3 Composition API + Tailwind CSS v4 + TypeScript，不能引入新的重型UI框架
- **Business**: 保持现有功能完整性，不能破坏已有的用户操作流程
- **Dependencies**: 继续使用 lucide-vue-next 作为图标库，ECharts用于数据可视化

## Assumptions
- 现有设计方向（暖墨色+朱红+衬线字体的手稿馆主题）是正确的，只需要深化和增强，不需要完全推翻重设计
- 用户在1440p及以上分辨率的桌面显示器上使用为主
- 可以使用CSS高级特性（backdrop-filter, mask, gradient, animation等）
- Source Serif 4 等字体可以通过适当方式加载或使用优雅降级方案

## Acceptance Criteria

### AC-1: 沉浸式视觉氛围
- **Given**: 用户打开应用
- **When**: 页面加载完成
- **Then**: 呈现出层次丰富、质感细腻的手稿馆氛围，包括可见但不刺眼的纸张纹理、边缘暗角、微妙的墨色渐变，整体感觉像走进一个温暖的古籍创作室
- **Verification**: `human-judgment`
- **Notes**: 纹理和效果不能影响内容可读性

### AC-2: 精致的侧栏导航
- **Given**: 用户在任何页面
- **When**: 查看左侧导航栏
- **Then**: 品牌标识区有辨识度，导航项有精致的激活状态（包括左侧指示条、背景、图标变化），组标签清晰优雅，底部控制区（主题/语言切换）设计统一
- **Verification**: `human-judgment`

### AC-3: 高品质组件系统
- **Given**: 用户与界面元素交互
- **When**: 悬停按钮、聚焦输入框、查看卡片
- **Then**: 每个组件都有微妙而精致的状态变化，按钮有按压/悬停的深度感，输入框聚焦时有优雅的光效，卡片有自然的阴影和边框变化
- **Verification**: `human-judgment`

### AC-4: 流畅的动效体验
- **Given**: 用户在页面间导航或触发交互
- **When**: 页面切换、元素出现/消失、按钮点击
- **Then**: 动效流畅自然（使用editorial缓动曲线），页面内容有阶梯式入场效果，通知有优雅的滑入滑出，没有生硬的瞬间跳变
- **Verification**: `human-judgment`
- **Notes**: 动效要有意义，纯粹装饰性的动效要克制

### AC-5: 仪式感的首页
- **Given**: 用户进入项目列表页
- **When**: 首次打开或返回项目页
- **Then**: 有欢迎性的品牌展示区域，项目列表卡片设计精致，新建项目按钮有引导性
- **Verification**: `human-judgment`

### AC-6: 沉浸式写作界面
- **Given**: 用户进入写作页面
- **When**: 进行写作操作
- **Then**: 写作区域有足够的留白和舒适的行宽，预览区域排版优雅像真正的手稿，控制区不干扰创作沉浸感
- **Verification**: `human-judgment`

### AC-7: 优雅的数据展示
- **Given**: 用户查看质量分析报告
- **When**: 查看各项分析指标和图表
- **Then**: 数据卡片设计有层次，图表配色与主题统一，指标徽章精致，整体不像技术仪表盘而像手稿评注
- **Verification**: `human-judgment`

### AC-8: 主题切换一致性
- **Given**: 用户切换墨色/纸色主题
- **When**: 点击主题切换按钮
- **Then**: 切换有平滑过渡，浅色主题同样具有高质量的纸张质感和温暖氛围，不是简单的颜色反转
- **Verification**: `human-judgment`

### AC-9: TypeScript类型安全
- **Given**: 完成所有代码修改
- **When**: 运行 `vue-tsc --noEmit`
- **Then**: 无类型错误
- **Verification**: `programmatic`

### AC-10: 构建成功
- **Given**: 完成所有代码修改
- **When**: 运行 `pnpm build`
- **Then**: 构建成功无错误
- **Verification**: `programmatic`

### AC-11: 现有测试通过
- **Given**: 完成所有代码修改
- **When**: 运行 `pnpm test`
- **Then**: 现有单元测试全部通过
- **Verification**: `programmatic`
