## Why

用户实际使用时，有两类中断场景极其常见：(1) API 额度耗尽（DeepSeek 返回 402/insufficient_quota），生成进度全部丢失；(2) 10 个场景全部执行时没有任何进度反馈，感觉像卡住了。当前 CLI 既没有保存每轮结果、也没有清晰的错误分类，更不支持从断点恢复。

## What Changes

- **场景流式显示**：CLI `_scene_execution_pipeline` 改用 `SceneEngine.run_scene_stream()`，每轮实时打印对话，不再阻塞等待
- **细粒度自动保存**：世界构建每轮后保存、角色生成后保存、剧情大纲后保存、场景每轮后保存进度
- **LLM 错误分类**：`LLMClient` 区分配额耗尽 (402/429)、网络错误、超时，返回结构化错误码
- **断点续传**：CLI 启动时检测未完成的会话，允许从中断的阶段精确恢复（而非只能从头开始世界构建）
- **恢复粒度**：世界构建→角色→关系→剧情→场景→写作，每阶段内也支持子步骤恢复
